# Почему Standalone Asio для Windows?

## Проблема с Boost на Windows

При использовании полного Boost на Windows через CMake FetchContent могут возникнуть проблемы:

1. **Большой размер**: Boost архив ~300MB, долго скачивается
2. **Долгая сборка**: Boost требует сборки нескольких библиотек
3. **Зависимости**: Требует правильной настройки toolchain (MinGW/MSVC)
4. **Конфликты версий**: Может конфликтовать с системными библиотеками

## Решение: Standalone Asio

**Standalone Asio** - это header-only версия Boost.Asio без зависимостей от остального Boost.

### Преимущества:

✅ **Header-only**: Не требует компиляции, просто include
✅ **Маленький размер**: ~10MB git репозитория
✅ **Быстрая загрузка**: Скачивается за секунды
✅ **Без зависимостей**: Работает сразу после скачивания
✅ **Кроссплатформенность**: Одинаково работает на Windows/Linux/macOS
✅ **100% совместимость API**: Тот же самый API, что и Boost.Asio

### Изменения в коде:

Вместо:
```cpp
#include <boost/asio.hpp>
using boost::asio::ip::tcp;
boost::asio::io_context io_context;
```

Используем:
```cpp
#include <asio.hpp>
using asio::ip::tcp;
asio::io_context io_context;
```

### Что нужно добавить:

1. **ASIO_STANDALONE** - макрос, указывающий что используется standalone версия
2. **ws2_32** - библиотека Windows Sockets (для сетевых операций на Windows)

Это всё! Код идентичен, но сборка проще и быстрее.

## Сравнение производительности

| Критерий | Boost.Asio | Standalone Asio |
|----------|-----------|-----------------|
| Загрузка зависимостей | ~5-10 минут | ~30 секунд |
| Размер загрузки | ~300 MB | ~10 MB |
| Время первой сборки | ~10-20 минут | ~1 минута |
| API | Полный | Полный (Asio) |
| Производительность runtime | Идентична | Идентична |

## Для других платформ

Standalone Asio отлично работает на всех платформах:

- **Windows**: Линкуется с `ws2_32`
- **Linux**: Линкуется с `pthread` (если нужно)
- **macOS**: Работает без дополнительных библиотек

Код остается одинаковым для всех платформ!
